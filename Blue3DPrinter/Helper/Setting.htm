Open the xml file Blue3DPrinter.exe.config with notepad or any text editor.The root directory is always the folder of Blue3DPrinter.exe

Set string for "GameDirectory"

The location of your game, example for me it's: "C:\Program Files (x86)\Steam\steamapps\common\Empyrion - Galactic Survival".
The program requires some files to work, currently the only file that it need is "BlockConfig.ecf". If you want to use a custom BlockConfig.ecf, put it in the program directory (where is Blue3DPrinter.exe), the program give it the priority.

 

Set string for "ModelsFolder"

Set as "ModelsFolder", it's the location of my modelStorage.data.

modelStorage.data:
 This file contains all the geometries exported from the bundle's asset files: models and shapes. The bundle models2 was been left out, contains the decoration's blocks, in fact the debug output of the tool will report some red messages for these missing block, they will be added in future releases. Since opening and reading the assets took about 5 minutes I decided to create my file storage to extract the geometries very fast

WARNING: if you delete this file the toolaskyou to rebuild it, but it take about 5min due to poor performance of AssetStudio library.
 To read the assets I used the library: "AssetStudio.dll" from https://github.com/Perfare/AssetStudio . It can read bundles files,it'snot fast but it'snecessaryto to extract the 3D geometries and build themodel.

Another solution could be the use of https://github.com/nesrak1/AssetsTools.NET. I tried to use it, it's very fast but it's too complicated.

 

Set string for " ModelsFolderFBX"

Set as "ModelsFolderFBX" the location of additional asset's fbx file to override those are in modelStorage.data (only for debug), but it's not implemented yet.

Set number for "HideFaceMethod"

The game's engine optimizes and remove the invisible triangles of the hull's blocks, i do the game... or at least it should.

0 = Don't hide invisible faces from hull block, extract all data. (for debug)
1 = Fast hide method that use the name of mesh (the game store a prefix inside mesh name)
2 = Slow hide method but hide all possibles faces, like in-game

Set number for "MaxVerticesPerObject"

The maximum vertices for each block (0 = no-limit), all blocks with more vertices will be omitted.
This is a temporary solution because some objects contain thousands of vertices and the file size becomes too large. This is not a problem for the tool which is capable of creating large files in some seconds, but if you try to convert a capital ship class "Fps-Destroyer", your 3d graphics programs will suffer a lot.The game's graphics engine instead uses different methods to reduce the number of vertices and be able to render in realtime.
